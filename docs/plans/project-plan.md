# 프로젝트 기획 · ai-news-supplier

> 상세 확정 계획: `docs/plans/2026-07-09-ai-news-supplier-plan.md` (이 문서는 요약·상위 기획)

## 문제 정의
- AI 도구·모델·기술이 빠르게 등장하지만, LLM 에이전트는 학습 시점 이후의 최신 맥락을 모른다. 사용자가 매번 직접 검색해 붙여넣지 않으면 에이전트가 최신 AI 소식을 활용할 수 없다.
- 최신 기술을 "소비"에서 "학습"으로 연결하는 경로도 부족하다.

## 목표와 성공 기준
- 성공 기준 1: `npx` 설치 후 API 키 없이 `ains trends`가 실제 최신 AI 소식을 출력한다.
- 성공 기준 2: Claude Code에 MCP 서버로 등록하면 에이전트가 `get_trends`, `search_news`로 최신 맥락을 조회할 수 있다.
- 성공 기준 3: `learn-today` 프롬프트로 에이전트가 핫한 토픽의 학습 세션을 설계·진행하고, 학습 이력이 기록·반영된다.
- 성공 기준 4: 한 수집 소스가 죽어도 나머지 소스와 전체 기능이 정상 동작한다.

## 범위
- 포함: 수집기 7종(HN, GitHub, 공식 RSS, Hugging Face, arXiv, DEV.to, Reddit[키 선택]), 로컬 SQLite 축적, 룰 기반 랭킹, MCP 도구 9종+프롬프트 3종, CLI, OS 스케줄러 등록(선택), 학습 후보 발굴·세션 설계·이력 관리
- 제외: 웹 UI, 자체 LLM 호출, 클라우드 호스팅(코어 분리로 전환 가능성만 확보), 외부 도구 쓰기 연동

## 주요 사용자와 사용 시나리오
- LLM 에이전트를 쓰는 개발자가 (1) 에이전트에게 "요즘 AI 뭐가 핫해?"라고 물으면 에이전트가 MCP로 조회해 답하고, (2) `/learn-today`로 오늘 학습할 토픽을 추천받아 에이전트 주도 학습 세션을 진행하고, (3) 터미널에서 `ains trends`로 직접 확인한다.

## 산출물
- npm 패키지 `ai-news-supplier` (bin: `ains`, `ains-mcp`), 오픈소스 공개(GitHub)

## 일정·마일스톤
- 마감: 없음(실사용 도구, 품질 우선·속도 중시)
- 주요 단계: S0 스캐폴드 → S1 워킹 스켈레톤(HN+CLI) → S2 수집기 완성 → S3 MCP 서버 → S4 학습 기능 → S5 스케줄러·배포 준비 (상세: docs/plans/implementation-plan.md)

## 리스크와 대응

- 룰 기반 학습 후보 품질 한계 → 엔티티 사전·블록리스트를 데이터로 관리해 조정 용이하게
